## 2. Найденные проблемы

**Критично (P0):**
1. Нет отдельных URL товаров — нельзя дать ссылку, SEO нулевое, промпт-требование №1.
2. Цена в ¥ как основная валюта — для казахстанского клиента непонятно (промпт §18: убрать).
3. Одно фото на товар без zoom — при 914 товарах с 2+ фото в МойСкладе.
4. Нет статуса наличия/срока — клиент не понимает «когда и как».
5. Контакты: только Telegram (промпт: убрать «Telegram как единственный контакт»).
6. Один язык (RU) — нет KK/EN.
7. 404 нет (Plesk отдаёт свою), редиректов нет.
8. Форма заявки: нет компании/города/способа связи, нет антиспама и защиты от двойной отправки, нет номера заявки.
9. `about.html` — «официальный представитель Zhongshi» — юридически неподтверждённая формулировка (промпт: заменить на редактируемую «казахстанское направление Zhongshi 中石»).

**Средне (P1):** нет хлебных крошек, сортировки, «похожих товаров», FAQ, страницы «Доставка и оплата», аналитики.

**Мелкое (P2):** личный кабинет, PDF-заявка, сравнение, кейсы.

## 3. Чего НЕТ в данных (не выдумываем — помечаем «Уточняется»)

| Требование промпта | Реальность МойСклада |
|---|---|
| Цвета/варианты, colorHex, свотчи | Данных о цветах НЕТ. Вариантность в МС — по размеру партии, не по цвету. → Архитектура поля заложена, UI появится после заполнения данных |
| Склад Казахстан / остатки | Склада в РК НЕТ (модель проекта: прямые поставки под заказ, решение CEO 17.07). Статус честный: «Под заказ с фабрики (Китай)» |
| Сроки поставки | Ставки/сроков карго нет (задача №1). → «Срок уточняется при расчёте» |
| priceTiers, палеты | Нет в данных |
| Кол-во в коробке | Атрибут `Qtt1Box` есть у ~1329 позиций МС — ДОСТАЁМ и показываем |
| Телефон/WhatsApp/email | У проекта пока только Telegram-бот. → редактируемый `data/site.config.json`, блоки скрыты до заполнения |
| Сертификаты, отзывы, кейсы | Нет. НЕ публикуем выдуманное (промпт §13, §18) |

## 4. План реализации P0 (наш стек: SSG → статика + PHP)

1. **SSG-генератор** `04_Scripts/site_v2/build.js`: из МойСклада/кэша →
   - `/{ru,kk,en}/index.html` — главная;
   - `/{lang}/catalog/` и `/{lang}/catalog/<категория>/` — каталог с фильтрами;
   - `/{lang}/product/<sku>/` — 620 страниц товара × 3 языка;
   - `sitemap.xml`, `robots.txt`, `404.html`, `.htaccess` (404 + 301 со старых `?cat=`).
2. **Галерея**: скачать до 4 фото на товар из МС; лайтбокс с zoom, стрелками, Escape, свайпом.
3. **Цены**: основная — **KZT** «от X ₸» по курсу НБ РК 69,05 (23.07.2026, зашивается с датой при сборке), ¥ — справочно мелким. Пояснение «без доставки, точный расчёт по заявке».
4. **Карточка/страница товара**: SKU, цена ₸, MOQ, кол-во в коробке (где есть), статус, характеристики (объём/стекло/форма/ролик), похожие товары той же категории.
5. **Корзина-заявка**: сохранить механику localStorage; форма + компания, город, способ связи, WhatsApp/Telegram; honeypot-антиспам; блокировка двойной отправки; номер заявки; менеджеру — точный список SKU (канал order.php → бот работает).
6. **Языки**: RU полный; KK/EN — интерфейс переведён, названия товаров fallback RU (машинный перевод товаров не публикуем — промпт §9); переключатель сохраняет страницу и корзину; hreflang ru-KZ/kk-KZ/en-KZ + x-default.
7. **Брендинг**: «Flacon.kz by Zhongshi 中石» — из `site.config.json` (редактируемо); юрформулировка «казахстанское направление Zhongshi» тоже.
8. **A11y/адаптив**: alt, focus-состояния, aria у галереи/фильтров, hit-area 44px, sticky CTA на мобильном товаре, без горизонтального скролла на 360–1440.

**Не входит в этот заход (нужны данные/решения CEO):** цвета-варианты (нет данных), склад РК (нет склада), priceTiers, CMS (управление = МойСклад, документируем), аналитика (P1), английские/казахские описания товаров (нужен переводчик/носитель).
